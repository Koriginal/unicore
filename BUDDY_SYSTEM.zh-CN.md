# 🐾 编程小伙伴（Buddy）系统说明指南

欢迎使用 UniCore 内置的互动宠物模块：**Buddy 编程小伙伴系统**。
在你写代码、思考、战斗和探险的过程中，它会陪你一起成长，还会随机冒出灵感气泡、掉落专属称号。

默认情况下 Buddy 是开启的。若你曾关闭过，可随时重新打开：

```bash
/buddy enable
```

如果想让它暂时休息（隐藏显示并停止互动气泡），可用：

```bash
/buddy disable
```

---

## 💬 核心指令一览（详细版）

### `/buddy`
**作用**：查看当前跟随宠物的完整资料卡。  
**你会看到**：
- 名字、性格、物种、稀有度、等级、当前 XP
- 当前 HP / SP
- 已获得称号
- 下一称号与进度百分比
- 五维属性条（PATIENCE/SNARK/WISDOM/DEBUGGING/CHAOS）

---

### `/buddy pet` 或 `/buddy 摸摸`
**作用**：日常互动，快速培养感情。  
**效果**：`XP +5`。

---

### `/buddy list`
**作用**：查看全部宠物列表（你的“宠物仓库”）。  
**你会看到**：
- 每只宠物的序号
- 物种、等级、稀有度
- 哪一只是 `ACTIVE`
- 最近获得的称号（若有）

---

### `/buddy swap [数字]`
**作用**：切换当前跟随宠物。  
**例子**：`/buddy swap 2`

---

### `/buddy rename [新名字]` 或 `/buddy 改名 [新名字]`
**作用**：改名。  
**两种模式**：
- 直接写名字：手动改名（如 `/buddy rename 破坏神`）
- 不写名字：AI 会重生成名字和性格文案

---

### `/buddy feed`
**作用**：投喂宠物。  
**效果**：
- `XP +15`
- `SP +20`（不超过最大 SP）
- 冷却时间：**5 分钟**

---

### `/buddy wild` 或 `/buddy encounter`
**作用**：野外遭遇战（回合制）。  
**可用操作**：
- `[A]` 普攻：基于 `SNARK` 与 `CHAOS` 计算伤害
- `[S]` 特攻：消耗 `20 SP`，基于 `WISDOM` 造成更高伤害
- `[D]` 防御：本回合有效减伤（已修复）
- `[C]` 捕捉：血线越低成功率越高
- `[T]` 换宠：切换到其他存活宠物
- `[R]` 逃跑：立刻撤退

**结算**：
- 捕捉成功：主宠 `XP +20`
- 击败野怪（未捕捉）：主宠 `XP +15`
- 主动逃跑：全队 `-10 XP`
- 全灭：全队 `-10 XP`

---

### `/buddy raid` 或 `/buddy boss`
**作用**：Boss 团战（不可捕捉）。  
**特点**：
- 首领血量极高，建议多宠轮换（`[T]`）
- 防御减伤有效

**结算**：
- 胜利：全队 `+100 XP`
- 逃跑或失败：全队 `-20 XP`

---

### `/buddy explore`
**作用**：放置探险。  
**流程**：
1. 第一次输入：派出探险（计时开始）
2. `3 分钟`后再次输入：结算并返回探险日志

**奖励（三档随机）**：
- 普通：`XP +30`，`HP +0`，`SP +4`
- 稀有：`XP +40`，`HP +4`，`SP +8`
- 神话：`XP +50`，`HP +8`，`SP +15`

---

### `/buddy digest` 或 `/buddy evaluate`
**作用**：代码试毒玩法。  
**流程**：随机抽取工作区代码片段，让宠物给出风格化点评。  
**效果**：`XP +25`。

---

### `/buddy fuse [序号1] [序号2]` 或 `/buddy 融合 [序号1] [序号2]`
**作用**：融合两只宠物（不可逆）。  
**机制概览**：
- 产物继承融合后的外观与属性骨架
- 产物会成为新的 ACTIVE
- 产物出生即满状态（HP/SP 按上限初始化）

---

### `/buddy free [数字]` 或 `/buddy release [数字]`
**作用**：放生宠物（不可逆）。  
**补偿机制**：当前主宠获得 `被放生等级 * 7` 的 XP。

---

### `/buddy mute` / `/buddy unmute`
**作用**：静音/取消静音 Buddy 互动气泡。  
**说明**：仅影响展示与互动提示，不会清空宠物数据。

---

### `/buddy enable` / `/buddy disable`
**作用**：Buddy 全局开关。  
**说明**：会持久化保存，重启后仍生效。

## ✨ 互动触发机制（怎么让宠物“说话”）

Buddy 会通过三条通道触发灵感互动：

### 1) 编码输入触发（`coding` / `question`）
在输入框打字时触发，典型条件：
- 距离上次此类触发至少约 `15s`
- 当前输入足够长（至少约 `18` 字符）且有实质新增
- 出现问号 `?`，或输入以 `\n / ) / ; / }` 结束

### 2) 思考停顿触发（`thinking`）
当你输入较长内容并停顿时，会在约 `7s`后尝试触发；同类触发有约 `30s`间隔。

### 3) 回合观察员触发（Observer）
每次模型回合结束后，Buddy 会按规则尝试旁观吐槽。若你在文本中直接提到宠物名字，触发概率会更高。

## 🏅 头衔系统（里程碑 + 随机掉落）

头衔来源有两种：

1. **里程碑解锁**：互动统计达到阈值自动解锁（coding/thinking/question/total/diversity）  
2. **随机掉落**：满足前置条件后，按权重从候选头衔里抽取

### 当前互动冷却（按类型）
- `coding`：15s
- `thinking`：30s
- `question`：12s

### 当前基础掉落概率
- `coding`：12%
- `question`：10%
- `thinking`：8%

### 保底机制
- 连续未掉落会累计保底计数
- 达到阈值后会触发保底掉落尝试
- 一旦掉落成功，计数归零

- Syntax Sniffer / Flow Channeler / Question Hunter / Pairing Oracle
- Bug Trapper / Mind Gardener / Architecture Owl / Soul Partner
- Stack Trace Diver / Lint Whisperer / Merge Conflict Tamer / Breakpoint Ranger
- Refactor Ritualist / Null Guardian / Cache Alchemist / Commit Bard
- Rubber Duck Archmage / Semicolon Sorcerer

## 📊 属性与恢复规则

### 基础公式
- `MaxHP = PATIENCE * 2 + 50`
- `MaxSP = WISDOM + 20`
- 升级需求：`lv * 20 + 20`

### 自动恢复（按时间差结算）
- HP：每 `30 秒 +1`
- SP：每 `45 秒 +1`
- 均不会超过上限

## ❓ 常见问题

### 1) 为什么宠物不说话？
先检查：
- 是否 `mute` 了
- 是否满足触发条件（输入长度、停顿、冷却间隔）
- 是否刚触发过同类互动（类型冷却未结束）

### 2) 为什么称号掉得慢？
建议混合触发 `coding / thinking / question` 三类互动，不要只刷单一路径。

### 3) 为什么我看不到边框或资料卡信息变少？
这通常是 UI 进入了简化展示分支，不是宠物数据丢失；重启 UI 后再看，或检查最近的启动配置。
